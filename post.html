OPERAZIONE: SCRUTINIO

Autore: Mirko | Data: 12 Apr 2026 | Status: Pubblicato

Durante un'analisi della rete scolastica abbiamo individuato un database accessibile senza autenticazione adeguata. I dati degli studenti risultavano esposti e venivano trasmessi verso indirizzi IP esterni non documentati.

Di seguito il log della scansione effettuata.

> nmap -sV 192.168.1.100

[*] Porta 3306/tcp aperta — MySQL 5.7

[*] Porta 80/tcp aperta — Apache 2.4

[!] Credenziali deboli rilevate

> SELECT * FROM sync_log LIMIT 3;

| 2026-04-10 | STU_0042 | 185.220.101.47 | READ |

| 2026-04-10 | STU_0087 | 185.220.101.47 | READ |

| 2026-04-11 | ALL | 185.220.101.47 | EXPORT |

[✓] Report completato.

I dati di centinaia di studenti venivano esportati verso server esterni. Pubblichiamo questa documentazione per sensibilizzare sulla sicurezza informatica nelle scuole.

Cosa abbiamo trovato

  • Database MySQL accessibile senza autenticazione sicura
  • Tabella sync_log con export automatico verso IP esterni
  • Nessun log di accesso visibile agli amministratori scolastici
  • Dati di oltre 400 studenti esposti

Consigli

  • Non trasmettere dati sensibili su reti Wi-Fi pubbliche o scolastiche non protette
  • Segnala eventuali anomalie al responsabile informatico della scuola
  • Usa connessioni cifrate quando possibile
